马尔科夫模型常用语分析大量随机事件，随机事件的特点是一个离散事件发生之后，另一个离散事件将在前一个事件的条件下以一定的
概率发生．以天气的马尔科夫模型为例：
晴天-->70%晴天   -->20%多云   -->10%下雨
多云-->15%下雨   -->70%多云   -->15%晴天
下雨-->50%下雨   -->25%晴天   -->25%多云
需要注意的几个点：
１．任何一个节点引出的所有可能的总和必须等于１００%，无论是多么复杂的系统，必然会在下一步发生若干事件中的一个事件
２．虽然这个天气系统在任一时间都只有三种可能，但是你可以用这个模型生成一个天气状态的无限次转移列表
３．只有当前节点的状态会影响后一天的状态．如果你在＂晴天＂节点上，即使当前１００天都是晴天或雨天都没有关系，明天
晴天的概率还是７０％
４．有些节点可能比其他节点较难到达，这个现象的原因用数学来解释非常复杂，但是可以直观的看出，在这个系统中任意时间节点上，
第二天是＂雨天＂的可能性（指向它的箭头概率之后下雨１００％），比＂晴天＂和＂多云＂要小很多
很明显，这是一个很简单的系统，而马尔科夫模型可以演化成任意规模的复杂系统．实际上，谷歌的pagerank算法是基于马尔科夫模型，
把网站作为节点，入站／出站链接作为节点之间的连线．连接某一个节点的＂可能性＂表示一个网站的相对关注度．也就是说，如果我们的
天气系统表示一个微型互联网，那么＂雨天＂的页面等级相对比较低，而＂多云＂的页面等级相对比较高．
